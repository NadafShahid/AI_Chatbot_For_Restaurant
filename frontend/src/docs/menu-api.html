Restaurant API - Menu Management

This documentation covers the Menu Management APIs for the Restaurant Backend API. All endpoints are prefixed with /api/menu.

Base URL

http://localhost:3000/api/menu

Authentication

Currently, no authentication is required for these endpoints. In production, implement proper authentication middleware.

API Endpoints

POST/api/menu

Add a new menu item

Request Body

{ "name": "Margherita Pizza", "price": 12.99, "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true }
FieldTypeRequiredDescription
namestringYesMenu item name
pricenumberYesItem price
descriptionstringNoItem description
categorystringNoItem category (e.g., Pizza, Salad)
typestringNoItem type (food, drink)
availabilitybooleanNoItem availability (default: true)
image_urlstringNoURL to the item's image

cURL Example

curl -X POST http://localhost:3000/api/menu \ -H "Content-Type: application/json" \ -d '{ "name": "Margherita Pizza", "price": 12.99, "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true, "image_url": "/images/pizza.jpg" }'

Success Response (201)

{ "success": true, "message": "Menu item created successfully", "data": { "id": 11, "name": "Margherita Pizza", "price": "12.99", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true, "image_url": "/images/pizza.jpg", "created_at": "2024-01-15T10:30:00.000Z" } }

Error Responses

400 Bad Request: Missing required fields
500 Internal Server Error: Database error

GET/api/menu

Get all menu items with optional filters

Query Parameters

ParameterTypeDescription
categorystringFilter by category (e.g., Pizza, Salad)
typestringFilter by type (food, drink)
availablebooleanFilter by availability (true/false)

cURL Examples

# Get all menu items curl -X GET http://localhost:3000/api/menu # Get only food items curl -X GET "http://localhost:3000/api/menu?type=food" # Get available pizza items curl -X GET "http://localhost:3000/api/menu?category=Pizza&available=true"

Success Response (200)

{ "success": true, "message": "Menu items retrieved successfully", "data": [ { "id": 1, "name": "Margherita Pizza", "price": "12.99", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true, "image_url": "/images/pizza.jpg", "created_at": "2024-01-15T10:30:00.000Z" }, { "id": 2, "name": "Pepperoni Pizza", "price": "14.99", "description": "Pizza topped with pepperoni and cheese", "category": "Pizza", "type": "food", "availability": true, "image_url": "/images/pepperoni.jpg", "created_at": "2024-01-15T10:25:00.000Z" } ] }

GET/api/menu/:id

Get menu item by ID

Parameters

ParameterTypeRequiredDescription
idintegerYesMenu item ID

cURL Example

curl -X GET http://localhost:3000/api/menu/1

Success Response (200)

{ "success": true, "message": "Menu item retrieved successfully", "data": { "id": 1, "name": "Margherita Pizza", "price": "12.99", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true, "created_at": "2024-01-15T10:30:00.000Z" } }

Error Response (404)

{ "success": false, "message": "Menu item not found" }

PUT/api/menu/:id

Update menu item information

Parameters

ParameterTypeRequiredDescription
idintegerYesMenu item ID

Request Body

{ "name": "Margherita Pizza Special", "price": 13.99, "description": "Classic pizza with tomato sauce, mozzarella, basil, and special sauce", "category": "Pizza", "type": "food", "availability": true }

cURL Example

curl -X PUT http://localhost:3000/api/menu/1 \ -H "Content-Type: application/json" \ -d '{ "name": "Margherita Pizza Special", "price": 13.99, "description": "Classic pizza with tomato sauce, mozzarella, basil, and special sauce", "category": "Pizza", "type": "food", "availability": true }'

Success Response (200)

{ "success": true, "message": "Menu item updated successfully", "data": { "id": 1, "name": "Margherita Pizza Special", "price": "13.99", "description": "Classic pizza with tomato sauce, mozzarella, basil, and special sauce", "category": "Pizza", "type": "food", "availability": true, "created_at": "2024-01-15T10:30:00.000Z" } }

Error Responses

400 Bad Request: Missing required fields
404 Not Found: Menu item not found
500 Internal Server Error: Database error

Testing with Postman

  1. Import this API documentation or create requests manually
  2. Set base URL to http://localhost:3000
  3. Test each endpoint in order
  4. Verify response codes and data structure
  5. Test error scenarios (invalid IDs, missing fields)
  6. Test filtering with query parameters

Common Response Format

All API responses follow this structure:

{ "success": boolean, // true for success, false for error "message": string, // Response message "data": object|array // Response data (only on success) }

Error Handling

  • 400 Bad Request: Missing or invalid request data
  • 404 Not Found: Resource not found
  • 500 Internal Server Error: Server/database errors

Notes

  • Filtering is supported via query parameters for GET /api/menu
  • Prices are stored as decimal values in the database
  • Consider adding image upload functionality for menu items
  • Implement proper validation and sanitization for security
  • Add pagination for large menu lists in production
