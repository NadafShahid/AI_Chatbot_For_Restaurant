Restaurant API - Point of Sale (POS) Module

This documentation covers the Point of Sale (POS) APIs for the Restaurant Backend API. All endpoints are prefixed with /api/pos.

Base URL

http://localhost:3000/api/pos

Authentication

Currently, no authentication is required for these endpoints. In production, implement proper authentication middleware.

API Endpoints

POST/api/pos/order

Create a quick order directly, bypassing the cart system. This is intended for POS terminals where orders are immediately placed and often paid.

Request Body

{ "user_id": 1, "items": [ { "item_id": 1, "quantity": 2 }, { "item_id": 3, "quantity": 1 } ], "payment_method": "cash" }
FieldTypeRequiredDescription
user_idintegerYesThe ID of the user placing the order.
itemsarrayYesAn array of objects, each containing item_id (integer) and quantity (integer).
payment_methodstringYesThe method of payment (e.g., "cash", "card").

cURL Example

curl -X POST http://localhost:3000/api/pos/order \ -H "Content-Type: application/json" \ -d '{ "user_id": 1, "items": [ { "item_id": 1, "quantity": 2 }, { "item_id": 3, "quantity": 1 } ], "payment_method": "cash" }'

Success Response (201)

{ "success": true, "message": "Quick order created and closed successfully.", "data": { "id": 101, "user_id": 1, "total_amount": "25.50", "payment_method": "cash", "status": "delivered", "created_at": "2024-01-15T10:30:00.000Z" } }

Error Responses

400 Bad Request: Missing required fields, invalid item quantity, or empty items array.
404 Not Found: User not found or menu item not found/available.
500 Internal Server Error: Database error or other server issues.

GET/api/pos/orders/active

Retrieves all orders that are currently active (i.e., not yet 'delivered' or 'cancelled').

cURL Example

curl -X GET http://localhost:3000/api/pos/orders/active

Success Response (200)

{ "success": true, "message": "Active orders retrieved successfully.", "data": [ { "id": 102, "user_id": 2, "total_amount": "15.00", "payment_method": "card", "status": "pending", "created_at": "2024-01-15T11:00:00.000Z", "user_name": "Jane Smith" }, { "id": 103, "user_id": 1, "total_amount": "30.25", "payment_method": "cash", "status": "preparing", "created_at": "2024-01-15T10:45:00.000Z", "user_name": "John Doe" } ] }

Error Responses

500 Internal Server Error: Database error or other server issues.

PUT/api/pos/orders/:id/close

Marks an existing order as 'delivered' and implicitly 'paid'. This is used to finalize an order from the POS perspective.

Parameters

ParameterTypeRequiredDescription
idintegerYesThe ID of the order to close.

cURL Example

curl -X PUT http://localhost:3000/api/pos/orders/102/close

Success Response (200)

{ "success": true, "message": "Order 102 closed successfully.", "data": { "id": 102, "user_id": 2, "total_amount": "15.00", "payment_method": "card", "status": "delivered", "created_at": "2024-01-15T11:00:00.000Z" } }

Error Responses

400 Bad Request: Invalid order ID provided.
404 Not Found: Order not found.
500 Internal Server Error: Database error or other server issues.

Testing with Postman/cURL

  1. Ensure the backend server is running.
  2. Use the provided cURL examples or Postman to test each endpoint.
  3. Verify response codes and data structure.
  4. Test error scenarios (e.g., invalid order ID, non-existent user/menu item).

Common Response Format

All API responses follow this structure:

{ "success": boolean, // true for success, false for error "message": string, // Response message "data": object|array // Response data (only on success) }

Error Handling

  • 400 Bad Request: Missing or invalid request data.
  • 404 Not Found: Resource not found.
  • 500 Internal Server Error: Server/database errors.

Notes

  • The POST /api/pos/order endpoint assumes immediate payment and marks the order as 'delivered' upon creation. Adjust this logic in posController.js if a different workflow is required (e.g., 'pending' status initially).
  • Consider adding authentication and authorization middleware for production use.
  • Implement robust input validation and sanitization for security.
