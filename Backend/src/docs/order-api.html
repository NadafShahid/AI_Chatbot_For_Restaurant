Restaurant API - Order Management

This documentation covers the Order Management APIs for the Restaurant Backend API. All endpoints are prefixed with /api/orders.

Base URL

http://localhost:3000/api/orders

Authentication

Currently, no authentication is required for these endpoints. In production, implement proper authentication middleware.

Order Status Flow

Orders follow this status progression:

  • pending → accepted → preparing → ready → delivered

Orders can only be cancelled when status is 'pending'.

Features

  • Create orders from cart items
  • Automatic cart clearing after order creation
  • Order status management
  • Order filtering by status and user
  • Order cancellation (pending orders only)
  • Complete order details with items

API Endpoints

POST/api/orders

Create order from cart items and clear cart

Request Body

{ "user_id": 1, "payment_method": "online" }
FieldTypeRequiredDescription
user_idintegerYesUser ID
payment_methodstringYesPayment method (e.g., 'online', 'cash', 'card')

cURL Example

curl -X POST http://localhost:3000/api/orders \ -H "Content-Type: application/json" \ -d '{ "user_id": 1, "payment_method": "online" }'

Success Response (201)

{ "success": true, "message": "Order created successfully", "data": { "id": 1, "user_id": 1, "total_amount": "28.97", "payment_method": "online", "status": "pending", "created_at": "2024-01-15T10:30:00.000Z", "user_name": "John Doe", "user_email": "john@example.com", "items": [ { "item_id": 1, "quantity": 2, "price": "12.99", "name": "Margherita Pizza", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food" }, { "item_id": 6, "quantity": 1, "price": "2.99", "name": "Coca Cola", "description": "Classic cola drink", "category": "Beverages", "type": "drink" } ] } }

Error Responses

400 Bad Request: Missing required fields or cart is empty
404 Not Found: User not found
500 Internal Server Error: Database error

GET/api/orders

Get all orders with optional filters

Query Parameters

ParameterTypeRequiredDescription
statusstringNoFilter by order status (pending, accepted, preparing, ready, delivered)
user_idintegerNoFilter by user ID

cURL Examples

# Get all orders curl -X GET http://localhost:3000/api/orders # Get orders by status curl -X GET "http://localhost:3000/api/orders?status=pending" # Get orders by user curl -X GET "http://localhost:3000/api/orders?user_id=1" # Combined filters curl -X GET "http://localhost:3000/api/orders?status=preparing&user_id=1"

Success Response (200)

{ "success": true, "message": "Orders retrieved successfully", "data": [ { "id": 1, "user_id": 1, "total_amount": "28.97", "payment_method": "online", "status": "pending", "created_at": "2024-01-15T10:30:00.000Z", "user_name": "John Doe" }, { "id": 2, "user_id": 2, "total_amount": "15.99", "payment_method": "cash", "status": "preparing", "created_at": "2024-01-15T11:00:00.000Z", "user_name": "Jane Smith" } ] }

PUT/api/orders/:id/status

Update order status

URL Parameters

ParameterTypeRequiredDescription
idintegerYesOrder ID

Request Body

{ "status": "preparing" }
FieldTypeRequiredDescription
statusstringYesNew status (pending, accepted, preparing, ready, delivered)

cURL Example

curl -X PUT http://localhost:3000/api/orders/1/status \ -H "Content-Type: application/json" \ -d '{ "status": "preparing" }'

Success Response (200)

{ "success": true, "message": "Order status updated successfully", "data": { "id": 1, "user_id": 1, "total_amount": "28.97", "payment_method": "online", "status": "preparing", "created_at": "2024-01-15T10:30:00.000Z", "user_name": "John Doe", "user_email": "john@example.com", "items": [ { "item_id": 1, "quantity": 2, "price": "12.99", "name": "Margherita Pizza", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food" } ] } }

Error Responses

400 Bad Request: Invalid status
404 Not Found: Order not found
500 Internal Server Error: Database error

DELETE/api/orders/:id

Cancel order (only if status is 'pending')

URL Parameters

ParameterTypeRequiredDescription
idintegerYesOrder ID

cURL Example

curl -X DELETE http://localhost:3000/api/orders/1

Success Response (200)

{ "success": true, "message": "Order cancelled successfully" }

Error Responses

400 Bad Request: Order cannot be cancelled (not pending)
404 Not Found: Order not found
500 Internal Server Error: Database error

Common Response Format

All API responses follow this structure:

{ "success": boolean, // true for success, false for error "message": string, // Response message "data": object/array // Response data (only on success) }

Order Data Structure

The order data includes:

  • id: Unique order identifier
  • user_id: User who placed the order
  • total_amount: Total order amount (string format)
  • payment_method: Payment method used
  • status: Current order status
  • created_at: Order creation timestamp
  • user_name: User's name
  • user_email: User's email
  • items: Array of order items with details

Error Handling

  • 400 Bad Request: Missing or invalid request data, cart empty, invalid status, order not cancellable
  • 404 Not Found: Order not found, user not found
  • 500 Internal Server Error: Server/database errors

Notes

  • Creating an order automatically clears the user's cart
  • Order status must follow the defined flow
  • Only pending orders can be cancelled
  • All monetary values are returned as strings for precision
  • Order items include full menu item details
  • Consider adding order history and audit logs in production
  • Implement proper user authentication and authorization
  • Add payment processing integration for production use
