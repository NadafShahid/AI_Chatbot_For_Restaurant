Restaurant API - User Management

This documentation covers the User Management APIs for the Restaurant Backend API. All endpoints are prefixed with /api/users.

Base URL

http://localhost:3000/api/users

Authentication

Currently, no authentication is required for these endpoints. In production, implement proper authentication middleware.

API Endpoints

POST/api/users

Create a new user

Request Body

{ "name": "John Doe", "email": "john@example.com", "phone": "+1234567890", "role": "customer", "password": "password123" }
FieldTypeRequiredDescription
namestringYesUser's full name
emailstringYesUnique email address
phonestringNoPhone number
rolestringNoUser role (default: 'customer')
passwordstringYesUser password

cURL Example

curl -X POST http://localhost:3000/api/users \ -H "Content-Type: application/json" \ -d '{ "name": "John Doe", "email": "john@example.com", "phone": "+1234567890", "role": "customer", "password": "password123" }'

Success Response (201)

{ "success": true, "message": "User created successfully", "data": { "id": 6, "name": "John Doe", "email": "john@example.com", "phone": "+1234567890", "role": "customer", "created_at": "2024-01-15T10:30:00.000Z" } }

Error Responses

400 Bad Request: Missing required fields
409 Conflict: Email already exists
500 Internal Server Error: Database error

GET/api/users

Get all users

cURL Example

curl -X GET http://localhost:3000/api/users

Success Response (200)

{ "success": true, "message": "Users retrieved successfully", "data": [ { "id": 1, "name": "John Doe", "email": "john@example.com", "phone": "+1234567890", "role": "customer", "created_at": "2024-01-15T10:30:00.000Z" }, { "id": 2, "name": "Jane Smith", "email": "jane@example.com", "phone": "+1234567891", "role": "customer", "created_at": "2024-01-15T10:25:00.000Z" } ] }

GET/api/users/:id

Get user by ID

Parameters

ParameterTypeRequiredDescription
idintegerYesUser ID

cURL Example

curl -X GET http://localhost:3000/api/users/1

Success Response (200)

{ "success": true, "message": "User retrieved successfully", "data": { "id": 1, "name": "John Doe", "email": "john@example.com", "phone": "+1234567890", "role": "customer", "created_at": "2024-01-15T10:30:00.000Z" } }

Error Response (404)

{ "success": false, "message": "User not found" }

PUT/api/users/:id

Update user information

Parameters

ParameterTypeRequiredDescription
idintegerYesUser ID

Request Body

{ "name": "John Smith", "email": "johnsmith@example.com", "phone": "+1234567899", "role": "customer", "password": "newpassword123" }

cURL Example

curl -X PUT http://localhost:3000/api/users/1 \ -H "Content-Type: application/json" \ -d '{ "name": "John Smith", "email": "johnsmith@example.com", "phone": "+1234567899", "role": "customer", "password": "newpassword123" }'

Success Response (200)

{ "success": true, "message": "User updated successfully", "data": { "id": 1, "name": "John Smith", "email": "johnsmith@example.com", "phone": "+1234567899", "role": "customer", "created_at": "2024-01-15T10:30:00.000Z" } }

Error Responses

400 Bad Request: Missing required fields
404 Not Found: User not found
409 Conflict: Email already exists
500 Internal Server Error: Database error

DELETE/api/users/:id

Delete user by ID

Parameters

ParameterTypeRequiredDescription
idintegerYesUser ID

cURL Example

curl -X DELETE http://localhost:3000/api/users/1

Success Response (200)

{ "success": true, "message": "User deleted successfully" }

Error Responses

404 Not Found: User not found
500 Internal Server Error: Database error

Testing with Postman

  1. Import this API documentation or create requests manually
  2. Set base URL to http://localhost:3000
  3. Test each endpoint in order
  4. Verify response codes and data structure
  5. Test error scenarios (invalid IDs, missing fields, duplicate emails)

Common Response Format

All API responses follow this structure:

{ "success": boolean, // true for success, false for error "message": string, // Response message "data": object|array // Response data (only on success) }

Error Handling

  • 400 Bad Request: Missing or invalid request data
  • 404 Not Found: Resource not found
  • 409 Conflict: Email already exists
  • 500 Internal Server Error: Server/database errors

Notes

  • Passwords are stored as plain text in this demo. In production, implement password hashing.
  • Consider adding authentication middleware for production use.
  • Add input validation and sanitization for security.
  • Implement rate limiting to prevent abuse.
