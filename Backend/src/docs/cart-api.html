Restaurant API - Cart Management

This documentation covers the Cart Management APIs for the Restaurant Backend API. All endpoints are prefixed with /api/cart.

Base URL

http://localhost:3000/api/cart

Authentication

Currently, no authentication is required for these endpoints. In production, implement proper authentication middleware.

Features

  • Automatic cart creation for new users
  • Add, update, and remove items from cart
  • Calculate total price automatically
  • Return full item details (name, price, etc.)
  • Handle item availability validation

API Endpoints

POST/api/cart/:userId/add

Add item to cart (auto-create cart if doesn't exist)

URL Parameters

ParameterTypeRequiredDescription
userIdintegerYesUser ID

Request Body

{ "item_id": 1, "quantity": 2 }
FieldTypeRequiredDescription
item_idintegerYesMenu item ID
quantityintegerYesQuantity to add (must be > 0)

cURL Example

curl -X POST http://localhost:3000/api/cart/1/add \ -H "Content-Type: application/json" \ -d '{ "item_id": 1, "quantity": 2 }'

Success Response (200)

{ "success": true, "message": "Item added to cart successfully", "data": { "cartId": 1, "items": [ { "item_id": 1, "quantity": 2, "name": "Margherita Pizza", "price": "12.99", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true } ], "total": "25.98" } }

Error Responses

400 Bad Request: Missing required fields or invalid quantity
404 Not Found: User not found or menu item not found or not available
500 Internal Server Error: Database error

GET/api/cart/:userId

Get user's cart with items and total price

URL Parameters

ParameterTypeRequiredDescription
userIdintegerYesUser ID

cURL Example

curl -X GET http://localhost:3000/api/cart/1

Success Response (200)

{ "success": true, "message": "Cart retrieved successfully", "data": { "cartId": 1, "items": [ { "item_id": 1, "quantity": 2, "name": "Margherita Pizza", "price": "12.99", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true }, { "item_id": 2, "quantity": 1, "name": "Coca Cola", "price": "2.99", "description": "Classic cola drink", "category": "Beverages", "type": "drink", "availability": true } ], "total": "28.97" } }

Empty Cart Response

{ "success": true, "message": "Cart retrieved successfully", "data": { "cartId": 1, "items": [], "total": "0.00" } }

PUT/api/cart/:userId/update

Update item quantity in cart

URL Parameters

ParameterTypeRequiredDescription
userIdintegerYesUser ID

Request Body

{ "item_id": 1, "quantity": 3 }
FieldTypeRequiredDescription
item_idintegerYesMenu item ID
quantityintegerYesNew quantity (0 to remove item)

cURL Example

curl -X PUT http://localhost:3000/api/cart/1/update \ -H "Content-Type: application/json" \ -d '{ "item_id": 1, "quantity": 3 }'

Success Response (200)

{ "success": true, "message": "Cart item updated successfully", "data": { "cartId": 1, "items": [ { "item_id": 1, "quantity": 3, "name": "Margherita Pizza", "price": "12.99", "description": "Classic pizza with tomato sauce, mozzarella, and basil", "category": "Pizza", "type": "food", "availability": true } ], "total": "38.97" } }

Error Responses

400 Bad Request: Missing required fields or invalid quantity
404 Not Found: User not found
500 Internal Server Error: Database error

DELETE/api/cart/:userId/remove/:itemId

Remove item from cart

URL Parameters

ParameterTypeRequiredDescription
userIdintegerYesUser ID
itemIdintegerYesMenu item ID to remove

cURL Example

curl -X DELETE http://localhost:3000/api/cart/1/remove/1

Success Response (200)

{ "success": true, "message": "Item removed from cart successfully", "data": { "cartId": 1, "items": [], "total": "0.00" } }

Error Responses

500 Internal Server Error: Database error

  2. Set base URL to http://localhost:3000
  3. Test each endpoint in order
  4. Verify response codes and data structure
  5. Test error scenarios (invalid user IDs, item IDs, quantities)
  6. Verify automatic cart creation for new users
  7. Check total price calculations

Common Response Format

All API responses follow this structure:

{ "success": boolean, // true for success, false for error "message": string, // Response message "data": object // Response data (only on success) }

Cart Data Structure

The cart data includes:

  • cartId: Unique cart identifier
  • items: Array of cart items with full menu details
  • total: Calculated total price (string format)

Error Handling

  • 400 Bad Request: Missing or invalid request data
  • 404 Not Found: Menu item not found or not available
  • 500 Internal Server Error: Server/database errors

Notes

  • Carts are created automatically when a user adds their first item
  • Item availability is checked before adding to cart
  • Setting quantity to 0 in update removes the item from cart
  • Total price is calculated automatically and returned as a string
  • All cart operations return the updated cart data
  • Consider adding cart persistence across sessions in production
  • Implement proper user authentication and authorization
